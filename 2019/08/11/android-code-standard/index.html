注：Android 规范覆盖部分Java规范

3.1. 包

基础规则：小写&&单词连续无间隔&&反域名法

模版：com.x.y.z ——> 说明：com=1级包名=固定com
x=2级包名=个人/公司名
y=3级包名=应用名
z=4级包名=功能模块名
示例：com.pingan.patech.login

常见的4级功能模块名

功能模块 命名规则 功能模块 命名规则
Activity com.x.y.activity 基础共享 com.x.y.base
Service服务 com.x.y.service Adapter类 com.x.y.adapter
广播服务 com.x.y.receiver 数据库 com.x.y.db
模型类 com.x.y.model 工具 com.x.y.util
自定义View com.x.y.view/ui JavaBean com.x.y.bean

3.3 变量

基础规则：（名字/名字短语）&&小驼峰原则

即名称中第一个词字母小写，其余单词首字母大写 eg：mLoginModel

常见变量命名规则

非常量

类型 命名规则 示例
非公有非静态 m开头 private int mAge
静态 s开头 static int sYear
其他 小写字母开头 int age

常量

类型 命名规则 示例
常量 单词全部大写，单词之间下划线相连 public static final int SOME_CONSTANT

临时变量

类型 命名规则 示例
临时变量名 整型：i/j/k/m/n；字符型：c/d/e for（int i=0;i<len;i++）

3.4 方法

基础规则：（动词/动宾）&&小驼峰原则

即名称中第一个词字母小写，其余单词首字母大写 eg：getInfo

方法名

类型 命名规则 示例
初始化 initXX() initView()
方法返回值是boolean isXX()/checkXX() isExistent()
返回某个值 getXX() getUserName()
处理数据 handleXX() handleUserInfo()
展示/弹出信息 showXX()/displayXX() showLoading()
保存数据 saveXX() saveUserInfo()
重置数据 resetXX() resetPwd()
清除数据 clearXX() clearCache()
移除数据 removeXX() removeUserInfo()
绘制相关 drawXX() drawCircle()

即名称中第一个词字母小写，其余单词首字母大写 eg：userName

3.7 额外

全局通用规范：

  1. 代码中的命名，不能以”_”或”$” 开始或结束，不能采用拼音、英文混合或中文。
  2. 类中使用设计模式，类中需要体现出对应的设计模式，如：ProductFactory，TeacherProxy。
  3. 对于使用Intent传递数据，以“KEY_” 开头
  4. 常见单词缩写

使用单词缩写的原则：只使用约定俗成的单词缩写，严禁自由缩写单词
